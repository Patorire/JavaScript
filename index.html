JavaScript

¿Qué es JavaScript?

JavaScript es un lenguaje de programación de scripts (secuencia de comandos) que está orientado a objetos. JavaScript es un lenguaje interpretado, es decir, no hay compilación. El código fuente se mantiene sin cambios y si se desea ejecutar el código se debe proporcionar un intérprete que va a leer y realizar las acciones solicitadas.
JavaScript actualmente es principalmente utilizado en internet, junto con las páginas web HTML O XHTML. JavaScript está directamente incluido en la página web (o en un archivo externo) y mejora una página HTML, añade interacción del usuario, animación, ayudas a la navegación...
JavaScript es considerado un lenguaje del lado del cliente, en otras palabras, los scripts son ejecutados por el navegador del usuario (cliente). Esto se difiere de los llamados lenguajes de script del lado del servidor que son ejecutadas por el servidor web, siendo este el caso de lenguajes como PHP. Esto es importante porque el propósito de los scripts del lado del cliente y del lado del servidor no son lo mismo. Un script que está del lado del servidor se encargará de la creación de la página web que se envía al navegador. Este entonces mostrará la página a continuación, ejecutará secuencias de comandos del lado del cliente como JavaScript.

JavaScript está diseñado con la intención de ser usado junto con HTML, el lenguaje ha ido evolucionando desde entonces hacia otros destino. JavaScript es usado regularmente para hacer extensiones para diferentes programas, como los scripts en Lua o Python.
JavaScript también se puede utilizar para construir aplicaciones, por ejemplo Mozilla Firefox, la interfaz del navegador se crea con una especie de HTML llamado XUL y JavaScript que se utiliza para la animación de la interfaz. Otros programas también están basados en esta tecnología como TomTom HOME que se utiliza para la administraci´n de el navegador GPS TomTom a través del ordenador.

¿Cuál es su historia?

En 1995, Brendan Eich trabajaba en Netscape Communications Corporation, la compañía que publicó el famoso Netscape Navigator, entonces principal competidor de Internet Explorer. Brendan desarrolló Live Script, un lenguaje de script que se basa en el lenguaje Java, y que estaba destinado a ser instalado en los servidores desarrollados por Netscape. Netscape inició el desarrollo de una versión del cliente LiveScript, que pasó a llamarse JavaScript, en homenaje al lenguaje Java creado por Sun Microsystems.

En ese entonces el lenguaje Java era cada vez más popular y Brendan Eich, el padre de JavaScript, al llamarlo JavaScipt en lugar de LiveScript era una forma de publicidad de Java y JavaScript entre si.
JavaScript fue lanzado en diciembre de 1995 y estaba integrado en Netscape Navigator 2. El lenguaje al ser tan exitoso hizo que Microsoft desarrollara una versión similar llamada JScript, que se instalaba en Internet Explorer 3, en 1996. Netscape decidió enviar a su versión de JavaScript a ECMA International (European Computer Manufacturers Association) con la intención de que el lenguaje fuera normalizado, dicho de otra manera querían que se creara una referencia del lenguaje y que así pudiera ser utilizado por otras personas y embebidos en otro software. ECMA Interantional estandarizó el lenguaje con el nombre de ECMAScript. Desde ese momento las versiones de ECMAScript han evolucionado. La versión más conocida es utilizadoa en todo el mundo, esta es la versión ECMAScript 3, publicado en diciembre de 1999.

Comprensión de Scripts

Algunas secuencias de comandos están disponibles en un formato comprimido, es decir, todo el código se escribe como secuencia, no hay retornos de línea. Esto reduce considerablemente el tamaño de una secuencia de comandos y sirve para segurar que la página see carga más rápido. Hay programas para "comprimir" el código JavaScript. Pero si has olvidado un simple punto y coma el código comprimido no funcionará porque las instrucciones no están correctamente separadas.

Espacios

JavaScript no es sensible a los espacios. Esto significa que puedes alinear las instrucciones que quieras, siempre que no interfiera con la secuencia de comandos.

Comentarios

Los comentarios son anotaciones realizadas por el desarrollador para explicar el funcionamiento de un script, una instrucción o incluso un grupo de instrucciones. Los comentarios no interfieren con la ejecución de un script. Hay dos tipos de comentarios: los de fin de línea y los multilínea.

Donde colocar el código en la página

Los códigos JavaScript son insertados a través del elemento *boca de pez menor que* script *boca de pez mayor que*. Este elemento tiene un atributo de tipo que se utiliza para indicar el tipo de lenguaje que vamos a utilizar. En este caso utilizamos el dicho previamente de JavaScript, pero podría ser otra cosa como VBScript aunque es bastante raro.
En HTML 4 y XHTML 1.x el tipo de atributo es obligatorio. En cambio en HTML 5, no lo es.

Variables

Para declarar una variable primero debes pensar el nombre que deseas ponerle. Es importante saber que el nombre de una variable solamente puede contener carácteres alfanuméricos, es decir, letras de la A a la Z y números del 0 al 9, guión bajo (_) y el símbolo del dólar ($).

Cuáles son los tres tipos principales de JavaScript

  • Tipo numérico: representa cualquier número (entero, negativo, notación científica...). Para asignar un tipo numérico a una variable se debe declarar como: var "nombre que le quieras dar a la variable"= "número que quieras";. Por ejempo: var numero = 5;.
  • Cadenas de carácteres (string): Este tipo representa texto. Este se asigna de la siguiente forma: var "nombre que le quieras dar a la variable" = '"texto que quieras escribir"';. Por ejemplo: var texto = 'Hola mundo';, también se puede escribir entre comillas var texto2 = "Hello world";.
  • Booleanos: el tipo booleano permite dos estados, verdadero o falso. Estos se escriben de la siguiente manera: var "nombre de la variable" = true; var "nombre de la variable" = false;.

Operadores aritméticos

Una vez que se ha declarado una variable y se le asigna un valor podemos empezar a operar con esas variables. Estos son la base para todos los cálculos (operadores aritméticos).

Si se toma el operador + no solamente se pueden hacer sumas sinó que se puede hacer lo que se llama concatenación entre cadenas.

Concatenación

La concatenación es añadir una cadena al final de otra:
var hola = 'Hola', nombre = 'tu', resultado;
resultado = hola + nombre;
alert (resultado)//muestra: «Holatu»

Operadores lógicos

Los operadores lógicos reciben el nombre de "lógicos" debido a que funcionan con el mismo principio que una tabla electrónica de verdad. Estos son los tres operadores:

  • Operador Y (AND): Este operador satisface la condición true cuando dos valores son verdaderos. Si uno de ellos no lo es sera falso.
  • Operador O (OR): Este operador devolverá true si uno de los dos valores es verdadero.
  • Operador NO (NOT): Este operador se diferencia de los otro porque solo necesita un valor. Recibe el nombre de "NO" porque su función es la de invertir el valor, es decir, si el resultado de otro operador boolean es true este pasará a ser false y viceversa.

Bucles

Un bucle es una estructura condicional similar a la mencionada anteriormente excepto que esta tiene la función de repetir una serie de instrucciones.

Bucle while

Para ejecutar un bucle es necesario definir una condición, mientras que esta sea verdadera (true) el bucle se repetirá. Una vez la condición deje de cumplirse (false) el bucle se detiene: (ejemplo de sintaxis)
while (condicion) {
sentencia_1;
sentencia_2;
sentencia_3;
}

Como hemos dicho hay que fijar un punto para que el bucle no se repita indefinidamente:

var numero = 1;
while (numero <10) {
número++;
}
alert (numero) / / Muestra: "10"

En primer lugar el número es 1. Al llegar al bucle preguntará si el número es menor que 10. Como si que lo es, se ejecuta el bucle. En cada rotación pregunta lo mismo hasta que cuando el número es igual a 10 la condición <10 pasa a ser falsa y por lo tanto el bucle cesa.

Break

Hay una palabra que tiene la capacidad de romper el ciclo de una vez. Esta palabra clave es break.

var = nicks = ' ', nick;
while (true) {
nick = prompt ('Introduzca un nombre:');
if (nick) {
nicks += nick + ' '; / / Agrega el nuevo nombre y un espacio después
} else {
break; / / Salir del bucle
}
}
alert (nicks) / / Muestra los nombres

Do while

Este bucle es muy parecido al bucle while, la diferencia es que el bucle do while siempre ejecutara el código una vez y después revisará si cumple la condición para seguir o no. (Ejemplo de sintaxis)

do {
sentencia_1;
sentencia_2;
sentencia_3;
} while (condición);

Bucle for

El bucle for también es parecido al bucle while, pero su arquitectura parece complicada al principio. El bucle for es un bucle que funciona de forma simple, pero que a ojos de personas principiantes la sintaxis se ve compleja. (Ejemplo de sintaxis)

for (inicio; condicion, incremento) {
sentencia_1;
sentencia_2;
sentencia_3;
}

En los parentesis del bucle ya no se encuentra la condición, sino tres bloques: inicio, condición e incremento. Estos tres bloques se separan por un punto y coma. Por lo tanto el bucle tiene tres bloques que lo definen. El tercer bloque es el incremento que se utiliza para el incremento de una variable en cada repetición del bucle. Por lo tanto es muy útil para contar y repetir el bucle un número determinado de veces.
